# 🏥 마이메디 패스 (MyMedi Pass)

> **모바일 신분증과 탈중앙화 DID 기반의 개인 주도형 의료 데이터 플랫폼**
> 2026 블록체인 & AI 해커톤 (Track 2: MVP 모델 개발 및 시연 부문)

---

## 💡 프로젝트 개요 (Overview)

**마이메디 패스(MyMedi Pass)**는 환자가 여러 병원에 분산된 자신의 민감한 의료 데이터(진료 기록, 처방전, 소견서, 영수증)를 중앙 서버가 아닌 **환자 본인의 스마트폰 지갑에 직접 소유하고 통제**할 수 있도록 돕는 탈중앙화 헬스케어 플랫폼입니다. 

정부 모바일 신분증 연동을 통해 강력한 보안성과 신원 인증을 확보하고, 서류 없는 원스톱 실손보험 청구 생태계를 구축합니다. 나아가 블록체인 메인넷 상의 실제 진료 트랜잭션을 기반으로 조작이 불가능한 청정 병원 트렌드 정보인 **"Today's Ranking (실시간 인기세)"**를 제공합니다.

---

## 🚨 제안 배경 및 필요성 (Problem Statement)

1. **아날로그식 의료 행정의 비효율성:** 환자가 과거 진료 이력을 확인하거나 타 병원으로 서류를 이관할 때, 혹은 실손보험을 청구할 때 반드시 과거 방문했던 병원을 직접 찾아가 종이 서류를 발급받아야 하는 번거로움이 존재합니다.
2. **디지털 속의 아날로그 모순:** 현재의 모바일 보험 청구는 병원 창구에서 종이 영수증을 출력한 뒤, 이를 스마트폰 카메라로 다시 촬영하여 보험사 앱에 업로드하는 모순적인 단계를 거치며 연간 수억 장의 종이 낭비와 불필요한 행정 검증 비용을 발생시킵니다.
3. **신뢰할 수 없는 병원 평점 시스템:** 기존 포털 및 예약 앱의 리뷰는 대가성 마케팅이나 영수증 조작 브로커에 의해 왜곡될 우려가 커, 환자들이 진짜 신뢰할 수 있는 병원 정보를 얻기 어렵습니다.

## ✨ 핵심 기능 (Key Features)

### 🔒 1. 모바일 신분증 기반 간편 환자 접수 (`Omnione CX`)
* 앱 로그인 및 병원 현장 접수 시 표준 인증 창을 통해 정부 모바일 신분증 검증을 수행합니다.
* 불필요한 주민등록번호 입력이나 플라스틱 진료 카드 없이 1초 만에 안전하게 신원을 증명합니다.

### 🌐 2. 마이 의료 데이터 지갑 (`OpenDID`)
* 병원(Issuer)에서 진료가 끝나는 즉시 진료 기록과 영수증을 탈중앙화 자격증명(VC) 형태로 환자 스마트폰 지갑(Holder)으로 발급합니다.
* 데이터 주권(Data Sovereignty)을 환자에게 환원하여, 환자가 필요로 하는 기관(Verifier)에만 선택적으로 제출(Selective Disclosure)할 수 있습니다.

### ⛓️ 3. 무서류 원스톱 보험 청구 (`Omnione Chain`)
* 앱 내에서 청구 버튼 터치 시 Solidity 기반의 스마트 컨트랙트가 발동되어, 암호화된 진료 내역 원본 데이터가 보험사로 위변조 없이 즉시 다이렉트 전송됩니다.

### 📊 4. 조작 불가능한 실시간 인기세 ("Today's Ranking")
* 광고나 허위 리뷰를 배제하기 위해, 실제 모바일 신분증으로 접수하고 진료를 받아 블록체인 메인넷 상에 기록된 **유효 트랜잭션 빈도수**를 기반으로 실시간 병원 순위를 집계하여 투명한 메디 트렌드를 제공합니다.

### 🤖 5. AI 기술 융합 (OCR & 메디케어 피드)
* **AI OCR 파이프라인:** 플랫폼 비연동 기관의 과거 종이 서류를 촬영하면 AI OCR 엔진이 질병 코드, 금액 등을 정밀 추출하여 디지털 데이터(VC 포맷)로 자동 변환합니다.
* **AI 맞춤형 건강 리포트:** 축적된 개인 건강 기록(PHR) 빅데이터를 AI 알고리즘이 분석하여 사용자 맞춤형 예방 행동 수칙 가이드를 제공합니다.

## 🚀 MVP 구동 및 시연 시뮬레이터 (Google Apps Script)

현재 레포지토리에는 프론트엔드 인프라가 갖춰지기 전, 심사위원들에게 완벽한 데이터 파이프라인 연동 로직을 증명하기 위해 **구글 스프레드시트 기반의 백엔드 시뮬레이터**가 구축되어 있습니다.

### 시뮬레이터 데이터베이스 구조
* **`User_Auth`**: `Omnione CX` 검증 서버를 거쳐 반환된 JWT 토큰 및 사용자 매핑 로그 기록
* **`Medical_Wallet`**: `OpenDID` 규격의 암호화 진료 데이터(VC) 발급 및 스마트 컨트랙트 기반 보험사 청구 이력 관리
* **`Today_Ranking`**: 메인넷 상의 실제 유효 진료 트랜잭션 숫자를 실시간 스케줄링 카운트하여 순위를 자동 변동시키는 대시보드

### 시연 메뉴 가이드
구글 스프레드 시트 상단의 **[🚀 마이메디 패스 MVP 시연]** 맞춤형 메뉴를 통해 다음 시나리오가 순차적으로 구동됩니다.
1. **1단계 (Omnione CX):** 사용자 성명 입력 시 정부 검증 서버 로직을 모의하여 암호화된 DID 식별자(`did:omnione:cx:...`) 자동 생성 및 적재
2. **2단계 (OpenDID):** 가상 병원 노드로부터 진료코드와 영수증 데이터를 디지털 자격증명(VC) 형태로 암호화 발급하여 유저 지갑에 안착
3. **3단계 (Omnione Chain):** 청구 실행 시 스마트 컨트랙트를 거쳐 고유의 블록체인 트랜잭션 해시값(`0x...`)을 실시간 생성하고 보험 청구 상태를 `COMPLETED`로 전환
4. **실시간 인기세 업데이트:** 트랜잭션 발생과 동시에 데이터 정렬 알고리즘이 작동하여 `Today_Ranking` 대시보드의 병원 순위 및 AI 트렌드 상태를 실시간으로 갱신

## 📈 기대 효과 (Expected Effects)

* **환자 중심:** 불필요한 병원 재방문 및 대기 시간을 제로화하고, 본인의 민감 정보 유출 우려 없이 안전하게 의료 선택권과 데이터 통제권을 행사합니다.
* **병원 및 보험사:** 제증명 서류 발급에 드는 창구 행정 피로도를 급감시키고, 위변조가 불가능한 원본 데이터를 통해 보험 지급 심사 업무를 실시간 자동화 궤도에 올립니다.
* **정부 및 사회적 가치:** 행정안전부가 주도하는 모바일 신분증의 사용처를 전 국민 체감 도메인인 '의료/보험'으로 확장하여 국가 디지털 인프라의 대중 활성화를 전력 견인합니다.
